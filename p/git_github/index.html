前言

  • Git：一个版本管理工具。它提供了类似类比于游戏存档的功能。有了存档，我们就可以方便地保存进度，在不同的节点之间转移。

  • Github: Github为你提供了存放Git仓库的服务器。你可以把自己的仓库上传到Github，就像保存存档到云端；你也可以拉取别人的仓库，就像获得了别人的游戏存档。所以Github同时也是一个强调开源分享的社区，你可以在Github上找到许多有趣的分享。

如果你是CS(Computer Science)相关专业的学生，git与github是你必须掌握的技能。如果你对Github社区或是我们的工作感兴趣，你也可以阅读这篇文章了解使用git工作的简单流程。此文会提供简单的实验，通过完成实验，你可以直接参与到网页的搭建中并留下你的名字。

让我们开始吧。

git init命令，即初始化命令，意味着我们要将此文件目录下的内容交给git管理。我们得到的输出Initialized empty repository in …意味着git已经帮你在该目录下构建了一个管理用的git仓库。

branch,Head

现在我们一共有两个commit，在第一个commit中，demo1.txt被写入了how r u，在第二个commit中,demo1.txt被写入了how old r u。我们按先后将它们记作commit-0和commit-1，有关系图如下：

commit-0 <- commit-1
                ^
                |
      Head -> master(main)

你也许会问，为什么是指向前一个commit的箭头呢？你可以理解为：每一个存档被保存时记录的是自己过去游玩到了什么进度，也方便回档重来。多个commit的存储方式类似于一个前向链表。

你也许还要问，master是什么？简单来说，master像一个指针，它总是指向一个branch（分支）的末尾，这个分支就名为master分支（或者叫main分支）。

那么head呢？它也像一个指针，指向一个branch或直接指向一个commit，它默认指向master branch。

让我们创建一个分支来解释这一切吧！

还记得先前提到的hash值吗？首先执行

1

git checkout 复制你的第一次commit的hash值到此

你会得到如下的段落，不要害怕，让我告诉你发生了什么

1
2
3
4
5
6

Note: switching to 'f58fa267f22a423518'.

You are in 'detached HEAD' state. You can look around, make experimental
changes and commit them, and you can discard any commits you make in this
state without impacting any branches by switching back to a branch.
略略略

在执行了checkout命令后，Head脱离master，直接指向了命令中指定的hash值对应的commit，即：

 commit-0 <- commit-1               
   ^           ^
   |           |
  Head       master

这样我们就来到了commit-0(first commit)。

Head此时指向commit-0，如果你继续提交新的commit,Head会接着指向新的commit，这个新的commit接在commit-0之后，也就是一个新的分支，但由于你没有给这个分支命名（没有指定一个标记了这个分支的指针），当你再一次使用checkout命令使Head去到另一个commit上时，git就会丢失你新提交的全部commit。这就是所谓的detached HEAD。

还记得吗？first commit中，我们在demo1.txt中写入了how r u（“怎么是你”）并提交，demo2.txt没有文字；second commit中，我们在demo1.txt中写入how old r u(“怎么老是你”)，demo2.txt也没有做任何回应，肯定是be了。呜呜呜，让我们从现在所处的commit 0开始，创建一个有he的分支吧Q_Q

执行！

1

git branch he

创建了一个由he指向的新分支，即：

commit-0 <- commit-1
 ^  ^          ^
 |  |          |
 he Head      master

记得上文所说的吗，如果你没有给新的分支命名，你在Head基础上提交的全部commit都会最终丢失，不仅如此，现在你为新分支指定了一个he指针，你就应该使Head指向he指针，通过he指针操纵新分支。也就是说，Head被设计为用来在各个commit间跳转，如果要构建分支，请使Head属于分支指针。你也许注意到了，之前的log输出中，有Head->master的字样。

所以接着执行

1

git checkout he

即：

commit-0 <- commit-1
 ^              ^
 |              |
 he <- Head   master

如果你不执行这条命令，会导致he不被你操作，留在原地，这样新的commit仍然不属于he分支。

打开demo2.txt，写入 im fine how r u

执行

1
2

git add .
git commit -m "he0"

git add . 命令中的点注意用英文标点，点表示该目录下的全部文件

把第三次commit记作commit-2，图示有：

commit-0 <- commit-1
   ^           ^
   |           |
commit-2     master
 ^    
 |    
 he <- Head

接着打开两个文档，写入 ^-^ 并保存，再执行

1
2
3

git add .
git commit -m "he1"
git log

把第4次提交记作commit-3，有图示：

commit-0 <- commit-1
   ^           ^
   |           |
commit-2(he0) master
   ^
   |
commit-3(he1)
 ^    
 |    
 he <- Head

好了，现在你可以使用git checkout 哈希值 在同一个分支上跳转了，如果你想要跳转到主分支，那就输入git checkout master即可。

你可能会发现，在跳转于各个commit途中，demo2.txt文档消失了，那是因为在master分支上，我们至始至终没有add过demo2.txt到git的管理范围，所以你跳转的这些快照中不存在demo2.txt。你可以通过在最开始执行

1
2

git add .
git commit -m "为工作区初始状态构建一个快照"

来构建一个你可以返回的初始状态。

Github

跟随以下的实验流程，在留言板上留言。

首先确保你安装了git，登录了github账号

点击这个链接访问实验用的Github仓库(https://github.com/Eureka-Emanon/Guestbook)，如果你无法直连，也不知道如何找代理，你可以使用edge浏览器访问此链接ilink并获取ilink一个可以免费代理Github的浏览器扩展，获取后点击并启用，再尝试访问Github仓库。

顺利访问我的Github仓库后，点击fork，这会将我仓库里的内容拷贝一份到你自己的仓库

点击fork后，不要更改仓库名，“fork the main branch only”默认勾选，点击Create fork，你的用户下就多了一个同名仓库。

在你的电脑上创建一个本地文件夹，取名为Guestbook，在该目录下打开cmd

cmd输入

1

git clone https://github.com/你的用户名/Guestbook

这条命令从Github上将你的Guestbook仓库克隆到本地，现在你的文件夹里应该有数据了
